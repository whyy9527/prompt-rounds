本轮从"用户旅程与路由架构"角度全面审查并重构小程序的页面跳转逻辑。

核心问题：当前页面各自为战，缺少对完整 user journey 的系统性设计。

请从上帝视角审查以下几条核心旅程是否顺畅、完整、无断点：

1. 新用户旅程：进入小程序 → 了解价值 → 授权登录 → 完成入会 → 到达首页
2. 点餐旅程：首页 → 选套餐 → 加入购物车 → 提交订单 → 结果反馈 → 下一步引导
3. 领小菜旅程：首页/会员页入口 → 查看可领内容 → 领取 → 成功反馈 → 回到主流程
4. 外卖返现旅程：入口 → 填写信息 → 提交 → 结果反馈 → 下一步引导
5. 邀请裂变旅程：入口 → 生成海报/链接 → 分享 → 进度查看 → 奖励到账反馈

重点检查：

- 每条旅程是否有清晰的起点、过程、终点
- 关键操作完成后是否有明确的"下一步去哪"
- 是否存在用户完成操作后迷失、回不去、不知道下一步的断点
- `app.json` 中页面路由定义是否完整、合理
- `wx.navigateTo` / `wx.redirectTo` / `wx.switchTab` 的使用是否符合场景语义
- tabBar 设计是否反映核心业务入口，而不是功能堆砌
- 深层页面是否有合理的返回路径和退出策略
- 是否存在循环跳转、死路、孤立页面

要求：

- 不要只改单页面，要从路由层面系统性重构跳转逻辑
- 直接修改 `app.json`、各页面的跳转调用和入口设计
- 每条旅程改完后说明：起点 → 路径 → 终点 → 下一步引导
- 以"第一次用这个产品的用户能不能顺畅走完"为标准
